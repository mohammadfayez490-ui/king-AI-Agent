# Database Schema

Purpose:

Define how AstraMind stores information.

Tables / Collections:

## Users

Stores:

- User ID
- Profile data
- Preferences
- Account settings


## Conversations

Stores:

- Conversation ID
- Messages
- Dates
- Context


## Projects

Stores:

- Project name
- Project data
- User ownership


## Memory

Stores:

- Approved memories
- User preferences
- Context information


## Settings

Stores:

- System configuration
- User options


Goal:

Create an organized database structure for AstraMind.
