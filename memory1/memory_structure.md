# Memory Structure

Purpose:

Define how AstraMind organizes saved information.

Memory Categories:

## User Memory

Stores:

- User preferences
- Communication style
- Selected settings
- Learning preferences


## Project Memory

Stores:

- Active projects
- Project details
- User goals
- Previous work context


## Conversation Memory

Stores:

- Important discussion context
- Relevant previous tasks
- Helpful history


## Control

Users can:

- View saved memory
- Remove memory
- Update preferences


Goal:

Create a safe and useful memory system for AstraMind.
